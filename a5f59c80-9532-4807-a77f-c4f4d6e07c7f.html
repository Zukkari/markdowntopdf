Queries

Queries 3 and 4 have been implemented via NativeQuery annotation. This was done because we first wrote the query in SQL and then when we tried to transform it into JPQL we faced some issues with query rewriting. This should not be a problem though, since all of the data processing and filtering is still done on database side.

As definition of query 4 was quite ambiguous, our interpretation is as follows:

  1. Query PlantInvItem
  2. Filter out reservations based on provided count and return count of reservations
  3. Based on remaining reservations, query maintenance tasks and return maintenance task count associated with those reservations.
  4. Group by results by PlantInvItem.id
  5. Order results by reservation_count and corrective_task_count
